## 2. 本地运行要求

后端要求：

- JDK 17
- Maven 3.9+
- MySQL 8
- Redis 6+

说明：

- 当前项目使用 `Spring Boot 3`
- `Spring Boot 3` 最低要求 `Java 17`
- 如果本机仍是 `Java 8`，项目无法正常编译

## 3. 当前环境已发现的问题

本机当前 `java -version` 为 `1.8`

因此执行 `mvn test` 时会报错：

- 类文件版本为 `61.0`
- 当前运行环境版本为 `52.0`

对应关系：

- `61.0` = Java 17
- `52.0` = Java 8

## 4. 推荐启动方式

### 4.1 切换到 JDK 17

确保 `JAVA_HOME` 指向 JDK 17，并重新打开终端。

### 4.2 初始化数据库

执行：

```sql
source backend/src/main/resources/db/schema.sql;
```

### 4.3 启动后端

在 `backend/` 目录执行：

```bash
mvn spring-boot:run
```

### 4.4 打开接口文档

启动成功后访问：

- `/swagger-ui.html`

### 4.5 默认后台账号

数据库初始化后默认账号：

- 用户名：`admin`
- 密码：`admin123`

### 4.6 启动后台前端

在 `admin/` 目录执行：

```bash
npm install
npm run dev
```

默认访问：

- `http://localhost:5173`

说明：

- 后台前端已通过 Vite 代理转发 `/api` 和 `/admin` 请求到 `http://localhost:8080`
- 本地开发时通常不需要再手动改前端接口地址
- 后台当前已支持订单、菜品、包间预约、宴席预约四个管理入口
- 后台当前已支持分类管理与菜品管理联动
