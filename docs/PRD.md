# 产品需求文档

## 1. 项目概述

- 项目名称：酒店二楼餐饮服务小程序
- 项目目标：为酒店二楼餐饮提供线上点餐、客房扫码送餐、包间预约、宴席预约能力
- 目标用户：普通顾客、酒店住客、包间预约客户、宴席咨询客户

## 2. 业务范围

### 2.1 第一阶段纳入范围

- 普通点餐
- 客房扫码点餐送房
- 包间预约
- 包间预点菜
- 宴席预约登记
- 商家后台订单管理

### 2.2 第一阶段暂不纳入

- 客房住宿预订
- 会员积分
- 优惠券营销
- 多门店
- PMS 深度对接

## 3. 用户角色

### 3.1 小程序用户

- 普通顾客：浏览菜单、下单支付
- 酒店住客：扫码识别房号后下单送房
- 包间客户：选择包间与时段并支付定金
- 宴席客户：提交婚宴、寿宴等预约需求

### 3.2 后台用户

- 管理员：拥有全量权限
- 运营人员：管理菜品、订单、预约
- 厨房/配送人员：处理制作与配送状态

## 4. 核心业务流程

### 4.1 普通点餐

1. 用户进入首页
2. 浏览菜品分类与详情
3. 加入购物车
4. 提交订单并支付
5. 商家接单制作
6. 订单完成

### 4.2 客房扫码点餐

1. 用户扫描房间二维码
2. 系统识别房号与送餐场景
3. 用户浏览菜品并下单
4. 商家接单与制作
5. 配送员送到房间
6. 用户确认收餐

### 4.3 包间预约

1. 用户选择预约日期
2. 选择早餐、中餐、晚餐时段
3. 选择包间
4. 填写人数与联系方式
5. 可选预点菜
6. 支付定金
7. 商家确认

### 4.4 宴席预约

1. 用户填写日期、人数、预算与需求
2. 提交预约
3. 商家后台跟进联系
4. 人工确认方案

## 6. 核心业务规则

### 6.1 客房扫码

- 一房一码
- 二维码绑定唯一 `qrCodeKey`
- 扫码后自动带出房号

### 6.2 包间预约

- 每个包间按早餐、中餐、晚餐三个时段预约
- 同一包间同一日期同一时段只能有一个有效预约
- 第一版支持固定定金

### 6.3 宴席预约

- 第一版只做预约登记
- 不在线锁库存
- 由后台人工跟进确认

### 6.4 订单取消

- 未支付订单 15 分钟自动关闭
- 已支付普通订单在接单前可申请取消
- 包间定金是否退还由商家规则决定

## 8. 成功指标

- 小程序完成上线并可正常下单
- 客房扫码点餐流程可闭环
- 包间预约可在线完成定金支付
- 宴席预约线索可在后台统一管理
